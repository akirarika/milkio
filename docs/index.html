In the past, our code used to run on a single server, which typically had many CPU cores and a large amount of memory. However, now our code often runs in individual Pods, which only have one or fewer CPU cores (yes, some Serverless providers even allow you to choose as little as 0.05) and limited memory. Even creating just one worker thread has a significant impact on cold start speed.

Since the lifespan of a Pod is typically short, loading all APIs into memory during startup, as we used to do, not only results in long startup times but also wastes a lot of memory. Milkio is only loaded when the API is actually called.
